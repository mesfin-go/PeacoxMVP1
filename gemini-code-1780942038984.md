# SYSTEM CONTEXT HANDOFF: Peacox Helping Hands MVP

## 1. Project Overview
**Name:** Peacox Helping Hands Portal
**Purpose:** A multi-role web application MVP designed to track, manage, and facilitate a youth mentorship and workforce empowerment program. 
**Roles:** Student (Mentee), Mentor, Program Manager (PM).

## 2. Tech Stack & Architecture
* **Frontend:** Pure Vanilla HTML5, CSS3, and JavaScript (ES6+). No frontend frameworks (React, Vue) or CSS libraries (Tailwind, Bootstrap) are currently used.
* **Backend / Auth:** Supabase (PostgreSQL). Currently utilizing `@supabase/supabase-js@2` via CDN.
* **Routing:** Vanilla JS `window.location` redirects based on Supabase Auth state and queried `profiles.role`.
* **State Management (Hybrid MVP State):** * *Authentication & Basic Profiles:* Handled via Supabase DB.
    * *Complex UI Data:* Currently utilizing browser `localStorage` as a temporary frontend database simulator (for Avatars, GAB metrics, Engagement selections, IDP goals, and Calendar notes). **Immediate Roadmap:** Migrate `localStorage` data structures to relational Supabase tables.

## 3. Core UI/UX Constraints & Design System
* **Color Palette:**
    * Primary: `--peacox-purple: #5A2E8E;`
    * Secondary: `--peacox-teal: #008080;`
    * Backgrounds: `--bg-color: #F8FAFC;`, `--card-bg: #FFFFFF;`
    * Text: `--text-main: #1F2937;`, `--text-muted: #6B7280;`
    * Status/Alerts: Green (`#10B981`), Amber (`#F59E0B`), Red (`#EF4444`), Brick (`#D16D54`).
* **The "No-Scroll" Rule:** Dashboards are designed with a strict `height: 100vh; overflow: hidden;` body. The main grid (`display: grid; grid-template-columns: repeat(4, 1fr)`) uses `min-height: 0; flex: 1;` to dynamically compress and fit the viewport exactly without vertical page scrolling.
* **Component Architecture:**
    * **Side Panes:** Slide-out right/left panels for user settings and data entry.
    * **Modals:** Centered overlay dialogues for form submissions and detailed views.
    * **Visualizations:** Pure CSS charts (Donut charts using `conic-gradient`, stacked horizontal bars, vertical flex bars, sparklines, and grid-based heatmaps). No 3rd-party charting libraries.

## 4. Supabase Database Schema (Current State)
* **Auth Table:** Standard Supabase `auth.users`.
* **`profiles` Table:** Trigger-linked to `auth.users`. 
    * Columns: `id` (UUID, PK), `first_name` (Text), `last_name` (Text), `role` (Text: 'student', 'mentor', or 'manager'), `avatar_initials` (Text).
* **Row Level Security (RLS):** Policies are assumed to be in place allowing users to read/update their own profiles.

## 5. Page-by-Page Feature Breakdown

### A. Front Door (`index.html`)
* **Function:** Unified login portal.
* **Features:** Bulletproof centering via Flexbox `min-height: 100vh`, custom password visibility toggle (SVG eye icon), authenticates via `supabaseClient.auth.signInWithPassword`, queries the `profiles` table for the user's role, and routes to the respective dashboard.

### B. Student Dashboard (`student.html`)
* **Layout:** 4x3 dynamic grid.
* **Features:**
    * **Header:** Dynamic Name, Active Status badge, Mentor assignment tag, Base64 Avatar Upload (saves to localStorage).
    * **Left Side Pane:** Navigation to data entry modals.
    * **GAB Tracker:** Modal to input Quarterly Grade, Attendance, and Behavior. Saves to local history array. Dashboard displays latest metrics + calculated % change (CSS arrow/color logic) from previous entry.
    * **Engagement Curriculum:** Modal with checkbox limiters (Max 2 Youth, Max 1 Comm, Max 2 Events). Synced dynamically to a styled dashboard card.
    * **IDP (Individual Dev Plan):** Goal setting and barrier identification. Short/Long term goals sync to the "Vision Statement" dashboard card.
    * **Calendar:** Minimalist interactive JS calendar for schedule/notes.

### C. Mentor Dashboard (`mentor.html`)
* **Layout:** 4x2 strict dynamic grid.
* **Features:**
    * **Header:** Dynamic profile details.
    * **Upcoming Sessions:** Compact card showing next 2 meetings.
    * **Mentee Roster:** Horizontal pulse-check cards showing mentee initials and status dots.
    * **Calendar:** Vertical interactive calendar (reused logic from student).
    * **Engagement Progress:** Boxed metrics showing X/Y completion fractions for caseload.
    * **GRO-WORK Pipeline:** Progress bars tracking mentee phases.
    * **Side Pane:** Settings for Basic Info, Mentor Profile (Industry/Topics), Capacity (Max mentees), and Ethics Code.

### D. Program Manager Dashboard (`pm.html`)
* **Layout:** 4x2 strict dynamic grid wrapped in a **Tabbed Interface**.
* **Features:**
    * **Header:** "Add System User" button (triggers `supabaseClient.auth.signUp` to bypass user-facing registration limits).
    * **Tab 1 - Cohort Overview:** CSS Sparklines for GAB trends, 3x2 Grid Heatmap for GAB Risk Matrix, IDP completion progress bars, Urgent Escalations list.
    * **Tab 2 - Mentor Operations:** Capacity Utilization gauge, Live Mentor Roster (compliance status/capacity), Mentor Escalations tracking, CSS Donut chart for Session Compliance.
    * **Tab 3 - Pipeline & Curriculum:** Global GRO-WORK funnel, CSS horizontal stacked-bar chart with legend for Youth Program distribution, Major Events RSVP tracker.
